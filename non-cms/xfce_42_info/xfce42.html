Some notes about Xfce 4.2

Here is a quick and dirty page about the development version of the Xfce desktop environment, the future 4.2.
The current stable version of Xfce is 4.0.5, and it's available from this page.

Here is the current schedule for Xfce 4.2 :

Aug 15th
  • Feature freeze
  • Messages freeze (so that translators can start working)
Aug 22th
  • Tag CVS for alpha 1 release
Week #35
  • Release Xfce 4.2 alpha 1 as source packages (possibly as RPMs if time permits)
Sep #19 (week #39)
  • Start xfce 4.2 beta release cycle
Nov 21st (week #48)
  • Start xfce 4.2 release candidate cycle

Notes: After Aug 15, no additions or change in existing messages is allowed.
Removal of messages may occur, when needed, as this doesn't require changes in po files.

How to test?

You can download and build the Xfce development version from CVS. An installation guide is available on this page. This version of course includes bugs, broken translations, out of date documentation, and may fail to build or even crash. If you find new bugs, please report them using the Xfce bug tracking system.

Xfce 4.2 will include many new features and several new applications.

List of the changes

Here is an incomplete list of the changes which are apparent for the user:

General
  • improved xinerama support
  • support for partial struts in xfwm4, xftaskbar4 and xfce4-panel.
  • freedesktop specifications compliance regarding menu and icons themes
xftaskbar4
  • "always group tasks" option
  • selectable left/center/right alignment
  • variable width
  • right-clic action menu
xfwm4
  • support for image composition with PNG images as layers, which means that one can do gradients or anything else, and that applies to any pixmap that make the window frame
  • native multihead support
  • several new themes
  • new themes location
  • supports the "show desktop" protocol
  • supports the KDE context help
  • "keep on top" option
  • icons in the "Alt+Tab" dialog
  • keyboard shorcuts editor included in the xfwm settings manager dialog
Settings manager
  • font rendering setup dialog
  • merged Desktop workspaces and Workspaces margins dialogs (so for a 4.0.x to 4.2 upgrade, you will need to remove workspaces_settings.* and margins_plugin.* from /usr/local/lib/xfce4/mcs_plugins/ or it won't work)
  • display resolution and gamma correction dialog
New session manager
  • Gnome/KDE compatibility
  • multiscreen safety
  • doesn't rely on smproxy (or any rewrite like gnome-smproxy), but provides (partial) session management even for non-sm or X11R5-sm aware apps
  • support for nice themes
xfdesktop
  • gradient support for background color
  • saturation support for background image
  • real multiscreen and xinerama support
  • optional support for icons in window list and desktop menu
  • desktop menu editor with DND support from xfce4-appfinder and .desktop files from Rox, Konqueror, Nautilus, Xffm,...
  • desktop menu plugin for the panel: now you can get standard menu from either the panel or the desktop
xffm
  • functional double pane
  • icons themes editor
  • many new features
panel
  • simplified mcs dialog
  • new positioning scheme
  • new panel icon theme implementation. The panel now follows the general icon theme, that can be set in the user interface dialog.
  • new way to add a plugin to the panel, using a dialog and mouse double-click
  • prettier move handles
  • unloading of unused modules
xfce4-mixer
  • complete rewrite
xfce4-appfinder
  • new Xfce4 application
gtk-xfce-theme-engine
  • new themes
