XFce 4是什么？

XFce 4是为类Unix系统设计的一个桌面环境。它的设计目标是在不失图形界面的美观、易用的同时提供一个快速、轻便的桌面环境。它是基于Gtk+开发库2.0版和http://freedesktop.org所定义的桌面环境标准开发的。

XFce 4包含了提供整个桌面环境功能的不同组件(群)。 基于组件(或叫模块)的设计可以让用户从整个发布中选择合适的组件(模块)组成最符合个人需要的桌面工作环境。

XFce 4 是完全免费(free)的软件。每个不同组件基于下列版权之一发布：BSD版权, GNU LGPL(GNU库版权) 或GNU GPL版权。

请访问XFce的网站http://www.xfce.org 以获取更多的信息。

新特征！

XFce 4.0-rc2 是新一代XFce桌面环境(4.0)的第二个正式发布候选版本，如果没有什么的问题这个版本将成为4.0的正式发布版。

XFce 4桌面环境的主要特征：

  • 基于GTK2: 提高了整体的显示效果及AA字体的支援。
  • 模块化的设计：使用中可以自由的选择合适的模块。
  • 提供开发平台支持：XFce 4提供了一套用来方便开发桌面应用程序的库，包括方便功能实现的工具、控件、对话框和全局的属性设置框架。
  • 拥有符合标准(Freedesktop定义的)的具有强大主题功能的窗口管理器，并随该窗口管理器一同发布了许多漂亮的主题。
  • 改进后能支持水平放置和垂直放置及自动隐藏、第三方插件支持等新功能的面板程序。
  • 整合SMB(用于和Windows机器现实文件共享的协议)共享浏览功能的文件管理器。
  • 整合全局属性设置、XFce组件及用户界面设置的设置管理器。
  • 许多附加功能的组件诸如任务条、图标盒子和打印控制对话框等。

同XFce 3相比，XFce 4完全重写了的全新版本，因此可以同时安装XFce 3和XFce 4两个桌面环境.

从RC1到RC2的修改

  • 修正了一个十分罕见窗口管理器相关的小错误。
  • 面板插件API针对旧版本插件“崩溃”的问题做了一些小的改动。旧版的插件将不能再在该版本的面板程序下工作。
  • 修正了图标盒子程序中的几个错误。
  • 一些翻译的更新。

BETA2到RC1的修改：

  • 许多内部库的改动。这就意味着连接libxfcegui4库的所有应用程序需要基于新库重新编译连接。
  • 面板程序添加了新的“分割线”项
  • “反转”功能项被从面板项对话框中去掉了。所有附加面板插件必须由作者做相应的更新。所有面板插件必须重新编译过。
  • 文件管理器新增可以拖拉重定位的标题栏。
  • 文件管理器新增具有通过Tab键补全功能的的输入框。
  • 窗口管理器在使用会话管理时具有了保存会话的功能。
  • 窗口管理器xrandr功能支持。
  • 增强了处理特殊窗口提示的能力。
  • 许多的错误修正。

BETA1到BETA2的修改：

  • Xinerama支持的功能加强。
  • 新的基于SVG图标的面板主题。
  • xfce4-extras添加了新的软件包：
    • xfce4-systray : 面板消息通知区插件。
    • xfce4-themes : 面板图标主题。
  • 默认情况下，使用Alt-Tab切换窗口输入焦点时，面板不会成为焦点窗口。
  • 面板程序允许使用目录名赋值给启动器来启动文件管理器(xffm)直接浏览该目录。
  • MCS设置使用了新的用户界面规划，比旧的更加美观。
  • xfce4-tips收入了几个有用的“技巧“。
  • 一些针对Solaris平台的错误修正。

软件包列表！

XFce 4.0-rc2发布包含下列软件包：

库

  • libxfce4util:图形界面无关的功能库。

  • libxfcegui4:控件和图形处理相关功能库。

  • libxfce4mcs: 为大部分XFce4模块使用的设置管理库。

桌面环境

  • xfce-mcs-manager: 设置管理器。为其他模块的设置插件提供一个设置集中管理器。S

  • xfce-mcs-plugins: 设置管理器插件。为不属于任何模块的设置项提供相应的设置对话框。当前，提供了用户界面(Gtk+)、鼠标、键盘的设置。

  • xfwm4: XFce4的窗口管理器。管理应用程序窗口，提供窗口修饰及工作空间管理。

  • xfce4-panel: XFce 4面板。提供程序启动器，工作空间切换器，时钟，面板菜单和其他更多的功能。

  • xfdesktop: 桌面背景管理器。该程序设置桌面背景并提供了桌面菜单，同时该程序还实现了桌面背景设置及增减工作空间的功能。

  • xfce-utils: 基础工具和脚本。提供包括任务条，关于对话框，运行程序对话框和其他一些供其他模块诸如面板程序使用的重要脚本，用户使用指南也在这中间。

  • xffm: 整合Samba网络共享浏览功能的文件管理器。

  • xfprint: 打印控制对话框。提供一个图形化的打印控制前端，包括xfprint4和xfprint-manager。

  • gtk-xfce-engine-2: 针对GTK2设计的主题引擎。本模块非桌面环境必须部分，但是这个漂亮的主题工具是值得一试的，它还自带了几套主题。

附加组件

  • xfwm4-themes: 窗口管理器的窗口修饰主题。

  • xffm-icons: xffm文件管理器的图标主题。

  • xfce4-themes: 面板程序的图标主题。

  • xfce4-iconbox: 图标盒子是xfce-utils工具包中提供的任务条程序的替代工具。

  • xfce4-mixer 面板的音量控制控件，包括一个简单的音量控制对话框。

  • xfce4-systray: 面板程序的消息通知区或叫系统消息盒。

  • xfce4-trigger-launcher: 表示两个不同状态(开/关)的特殊启动器插件。

  • xfce4-toys: 非常用工具盒面板插件。目前为止包含有“技巧”/“幸运一帖“程序，X Eyes程序的面板插件。

安装指南！

我们提供针对Redhat 8和9系统的RPM包。针对Debian系统的将很快出现在Debian的”unstable“版本中。

如果没能下载到针对你系统的二进制包或者需要自行编译所需软件包，请阅读下面的安装编译指南。

编译顺序

  • libxfce4util
  • libxfce4mcs
  • libxfcegui4
  • 桌面环境模块 (参见软件包列表)
  • 附加模块 (参见软件包列表)

编译一个单独的软件包其实很简单，解压缩源代码包，然后

进入源代码目录, 键入如下命令
./configure && make && make install

编译XFce 4需要pkg-config, GTK+ >= 2.0.6 (包括相应的开发包), libxml2 和 xffm需要的libdbh (>= 1.0_12), libdbh可以从 SourceForge下载。

作为可选项，要使用SVG图标需要安装librsvg 2 >= 2.2，而要使用”启动时通知“功能(启动程序时，鼠标变成等待状态)需要安装 libstartup-notification，以便支持相应的标准。

如果编译安装过程中使用不同的安装路径前缀(prefix)，还需要设置PKG_CONFIG_PATH环境变量，指向包含已安装XFce 4库的*.pc文件存放的路径。

例如，${prefix}/lib/pkgconfig
export PKG_CONFIG_PATH=/usr/local/lib/pkgconfig

所有的软件包都包含相应的spec文件以便用户可以以此编译他们

自己的RPM包。
rpmbuild -ta module.tar.gz

使用指南！

在命令行终端使用‘startxfce4’而非‘startx’来启动一个XFce 4会话。 如要定制startxfce4的启动过程，请复制 ${sysconfdir}/xfce4/xinitrc 到 /.xfce4/xinitrc, 增加该文件的可执行权限并按需要编辑该文件即可。

xfce-utils软件包为使用gdm的用户提供了便利脚本，如果在安装配置过程中侦测到gdm那么该脚本会被自动安装。 请参阅gdm，kdm或xdm文档关于如何添加为图形登陆管理器添加用户会话的章节以获取更多信息。 通常是创建~/.xsession文件并在该文件中调用startxfce，如此应该能通过图形登陆界面启动默认的会话了。~/.xsession文件也必须拥有可执行权限。

需要更多信息请参阅XFce 4文档和用户使用指南访问XFce 4文档的方法是使用命令‘xfhelp4’或从桌面菜单和面板右击菜单中选择‘帮助’菜单项。
